O treino que resolve o “deu branco”

Você vai treinar a habilidade que vem antes do código: olhar para o enunciado e montar a arquitetura mental da solução.

1O que entra? Dados digitados, argumentos, quantidade de valores, texto, nota, RA etc. 2O que guardar? Variáveis simples, arrays, matrizes, structs ou ponteiros. 3O que fazer? Soma, média, comparação, cadastro, impressão, alteração por referência. 4Tem decisão? if/else ou switch. 5Tem repetição? for, while ou do while. 6Tem função? Parâmetros, retorno, void e protótipo. 7O que sai? printf, relatório, lista, endereço, maior valor, resultado final.

Como usar

Faça os níveis em ordem. Cada nível tem três tipos de exercício:

🏷️ identificar blocos 🧩 ordenar receita ✍️ escrever esqueleto

O objetivo não é decorar uma solução. É reconhecer o tipo de problema e montar o caminho antes de escrever C.

Biblioteca rápida de gatilhos

Entrada e saída
  • leia, receba, informe -> scanf
  • mostre, imprima, exiba -> printf
  • valor digitado -> variável
Decisão e repetição
  • maior, menor, aprovado -> if
  • opção/menu -> switch
  • 10 valores -> for
  • até sair -> while/do while
Estruturas
  • vários do mesmo tipo -> array
  • linha/coluna -> matriz
  • nome + RA + nota -> struct
  • endereço/alterar original -> ponteiro
